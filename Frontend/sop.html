2

Candidate Entry (उम्मीदवार प्रविष्टि) नए उम्मीदवार का डेटा दर्ज करना

  • Sidebar से "Candidate Entry" पर क्लिक करें।
  • Candidate ID स्वतः generate होती है (UPP-XXXXX format)।
  • सभी required fields भरें: Merit No., Roll No., Reg. No., नाम, पिता का नाम, Mobile, District, Category।
  • Bulk Upload option से Excel file (.xlsx) द्वारा एक साथ अनेक उम्मीदवार जोड़े जा सकते हैं।
  • Sample Excel template download करने के लिए "Download Sample Excel" बटन दबाएं।
  • Submit करने पर उम्मीदवार database में Pending status के साथ save हो जाता है।
Excel Format: Sr. No. | Merit No. | Roll No. | Reg No. | Name | Father Name | Mobile | Email | District | Address | Selected As | 10th Status | 12th Status | Tech Status | Domicile Status | Caste Status | EWS Status
3

Candidate Edit (उम्मीदवार संपादन) पहले से दर्ज जानकारी सुधारना

  • Sidebar से "Candidate Edit" पर जाएं।
  • ID या नाम से Search करके उम्मीदवार को खोजें।
  • Edit (✏️) बटन से जानकारी update करें।
  • Delete (🗑️) बटन से उम्मीदवार का record स्थायी रूप से हटाएं (confirm करना होगा)।
4

Certificate Details (प्रमाण-पत्र विवरण) दस्तावेज़ सत्यापन स्थिति अपडेट करना

  • Sidebar से "Certificate Details" पर जाएं।
  • प्रत्येक उम्मीदवार के 6 प्रमाण-पत्रों की स्थिति अपडेट करें:
        • 10वीं प्रमाण-पत्र  • 12वीं प्रमाण-पत्र  • तकनीकी प्रमाण-पत्र
        • निवास प्रमाण-पत्र  • जाति प्रमाण-पत्र  • EWS प्रमाण-पत्र
  • Status Options: Verified / Pending / N/A
  • सभी 6 प्रमाण-पत्र Verified या N/A होने पर ही अगले चरण (Verification) की अनुमति मिलती है।
महत्वपूर्ण: जो प्रमाण-पत्र उम्मीदवार पर लागू नहीं होता (जैसे EWS General के लिए), उसे N/A करें।
5

Joining Letter Issue (नियुक्ति पत्र जारी करना) Verified उम्मीदवारों को Joining Letter देना

  • Sidebar से "Joining Letter Issue" पर जाएं।
  • यहाँ केवल Verified status वाले उम्मीदवार ही दिखेंगे।
  • "Issue Letter" बटन दबाने पर उम्मीदवार का issuedLetter = true हो जाता है।
  • "Print Letter" बटन से नियुक्ति पत्र print या PDF में save कर सकते हैं।
  • Letter issue होने की date स्वतः record होती है।
नोट: Joining Letter Issue के बाद ही उम्मीदवार Posting Allotment के लिए eligible होता है।
6

Posting Allotment (तैनाती आवंटन) उम्मीदवार को जिला आवंटित करना

  • Sidebar से "Posting Allotment" पर जाएं।
  • केवल Letter Issued उम्मीदवार ही यहाँ दिखेंगे।
  • "Assign District" बटन से उम्मीदवार को कोई भी UP जिला assign करें।
  • "District Capacities" पेज से यह देख सकते हैं कि किस जिले में कितनी सीटें बची हैं।
  • Allotment date स्वतः record होती है।
7

Dashboard एवं Report समग्र आँकड़े एवं फ़िल्टर रिपोर्ट

  • Dashboard पर कुल उम्मीदवार, Verified, Pending, Letter Issued और Posting Assigned के Summary Cards दिखते हैं।
  • Chart में Verification Status और District-wise वितरण देखें।
  • किसी भी Card पर क्लिक करने पर filtered report खुलती है (जैसे: सभी Verified उम्मीदवार)।
  • Report पेज पर Excel Export की सुविधा उपलब्ध है।

महत्वपूर्ण नियम एवं सावधानियाँ

  • Delete किया गया record पुनः प्राप्त नहीं होगा — सावधानी से Delete करें।
  • Bulk Upload में Duplicate Roll No. होने पर पुराना record overwrite हो जाएगा।
  • Certificate Details अपडेट करने के बाद ही Verify करें, अन्यथा Verification reject होगी।
  • System Internet connection के बिना काम नहीं करेगा — data cloud database (Supabase) में store होता है।
  • पहली बार login के बाद server start होने में 10-15 सेकंड लग सकते हैं — यह सामान्य है।
  • Session बंद करने से पहले Logout अवश्य करें।

1

Login Accessing the system

  • Open joining.shailbala-uppolice.shop in your browser.
  • Enter your Username and Password, then click "Login to Dashboard".
  • Username is case-insensitive — both admin and Admin are valid.
  • Invalid credentials will show an error message for 3 seconds.

Admin
Password

Note: On first login, the Dashboard may take 10–15 seconds to load as the server wakes up from sleep mode.
2

Candidate Entry Registering new candidates

  • Click "Candidate Entry" from the sidebar menu.
  • Candidate ID is auto-generated (format: UPP-XXXXX).
  • Fill all required fields: Merit No., Roll No., Reg. No., Name, Father's Name, Mobile, District, Category.
  • Use Bulk Upload to add multiple candidates at once via an Excel (.xlsx) file.
  • Click "Download Sample Excel" to get the correct template format.
  • On submission, the candidate is saved in the database with Pending status.
Excel Format: Sr. No. | Merit No. | Roll No. | Reg No. | Name | Father Name | Mobile | Email | District | Address | Selected As | 10th Status | 12th Status | Tech Status | Domicile Status | Caste Status | EWS Status
3

Candidate Edit Modifying existing records

  • Go to "Candidate Edit" from the sidebar.
  • Search candidates by ID or name using the search box.
  • Click the Edit (✏️) button to modify candidate information.
  • Click the Delete (🗑️) button to permanently remove a record (confirmation required).
4

Certificate Details Updating document verification status

  • Go to "Certificate Details" from the sidebar.
  • Update the status of all 6 certificates for each candidate:
        • 10th Certificate  • 12th Certificate  • Technical Certificate
        • Domicile Certificate  • Caste Certificate  • EWS Certificate
  • Available Status Options: Verified / Pending / N/A
  • All 6 certificates must be either Verified or N/A before final Verification is allowed.
Important: If a certificate is not applicable to a candidate (e.g., EWS for General category), mark it as N/A.
5

Joining Letter Issue Issuing appointment letters to verified candidates

  • Go to "Joining Letter Issue" from the sidebar.
  • Only candidates with Verified status will appear here.
  • Click "Issue Letter" to mark the candidate's letter as issued (issuedLetter = true).
  • Use "Print Letter" to print or save the letter as a PDF.
  • The letter issue date is automatically recorded in the system.
Note: A candidate becomes eligible for Posting Allotment only after their Joining Letter has been issued.
6

Posting Allotment Assigning candidates to a posting district

  • Go to "Posting Allotment" from the sidebar.
  • Only candidates whose Letter has been Issued will appear here.
  • Click "Assign District" to assign any UP district to the candidate.
  • Use "District Capacities" to check available seats in each district before assigning.
  • The allotment date is automatically recorded.
7

Dashboard & Reports Summary statistics and filtered reports

  • The Dashboard shows Summary Cards for Total Candidates, Verified, Pending, Letters Issued, and Posting Assigned.
  • Charts display Verification Status and District-wise distribution.
  • Click any card to open a filtered report (e.g., all Verified candidates).
  • The Report page supports Excel Export for offline use.

Important Rules & Cautions

  • Deleted records cannot be recovered — delete with caution.
  • If a Duplicate Roll No. is found during Bulk Upload, the existing record will be overwritten.
  • Always update Certificate Details before attempting Verification, otherwise it will be rejected.
  • The system requires an Internet connection — all data is stored in a cloud database (Supabase).
  • On first login, allow 10–15 seconds for the server to start — this is normal behaviour.
  • Always Logout before closing the browser session.
